# 输出格式规范

本文档说明 `idea-to-post` 技能的最终输出格式要求。

---

## Markdown 格式规范（必填）

所有文章内容必须以标准 Markdown 格式输出。

### 格式元素说明

| 格式元素 | 用途 | 示例 |
|---------|------|------|
| 标题层级 | 章节标题（仅二级） | `## 01｜引言` `## 02｜核心观点` |
| 列表 | 并列要点、步骤说明 | `1. 项目 1` |
| 代码块 | 统一使用 markdown 语言 | ` ```markdown ... ``` ` |

### 标题格式规范

统一格式：`## N｜标题`

1. 使用中文竖线 `｜`（不是英文竖线 `|`）
2. N 为数字序号（01/02/03...）
3. 标题简洁明了，能概括该部分内容

示例：
```markdown
## 01｜引言
## 02｜核心观点
## 03｜实践案例
## 04｜深度分析
## 05｜总结与展望
```

### 代码块规范

**统一使用 markdown 语言：**

````markdown
这里是代码内容
````

### 列表规范

统一使用有序列表：数字加点 `1.`
```markdown
1. 要点1
2. 要点2
3. 要点3
```

---

## 输出前检查清单

在最终输出前，必须确认以下事项：

### 格式检查

1. 标题格式统一：`## N｜标题`（使用中文竖线｜）
2. 代码块统一使用 markdown 语言
3. 整体格式遵循标准 Markdown 语法
4. 列表使用正确的符号（仅使用 `1.`）
5. 中英文之间有空格（AI Agent 而非 AIAgent）
6. 中文语境用全角标点（，。！？：；）

### 内容检查

1. 核心观点明确
2. 有真实案例（时间/地点/人物/数字）
3. 有情感共鸣（情绪时刻/身体反应/转折点）
4. 有独特观点（反直觉/未发现技巧/可引用原话）
5. 有原始素材引用（文档/对话/数据）
6. 故事闭环（全文及每个章节）
7. 段落不超过 5 行
8. 长短句交替

### 必填项检查（不可跳过）

1. 案例完整：至少 2 个具体案例
2. 情感完整：至少 2 个情绪时刻
3. 独特性：至少 1 个独特观点
4. 原始素材：至少 3 处直接引用
5. 故事闭环：全文有完整叙事弧

---

### 章节数量建议

1. 最少：3 个部分
2. 推荐：5 个部分
3. 最多：7 个部分

## 常见格式错误

### ❌ 错误示例

```markdown
# 一级标题（不应该使用）

## 1|引言（使用英文竖线）

### 三级标题（不应该使用）

- 要点1
- 要点2
（连续列表之间应该有空行）

```
代码块语言是 js（应该统一用 markdown）
```
```

### ✅ 正确示例

```markdown
## 01｜引言

正文内容...

## 02｜核心观点

1. 要点1

2. 要点2

```markdown
代码内容
```

## 语言风格规范

### 严禁使用的格式和词汇

**格式禁用：**
1. 严禁使用破折号（——）
2. 严禁使用粗体（**文字**）→ 用普通段落或中文标点「」代替强调
3. 严禁使用引用块（> 文字）→ 改为普通段落
4. 严禁使用无序列表（- 文字）→ 用编号列表或直接叙述

**重要说明：**
- 所有强调内容都用普通文本表达
- 标题层级已经足够，不需要粗体强调
- 引用内容直接融入段落，或用冒号引出
- 保持文章简洁自然，避免过度格式化

**词汇禁用（AI常用语气和词汇）：**
1. 本质上
2. 说白了
3. 不是...而是...
4. 众所周知
5. 综上所述
6. 总而言之
7. 由此可见
8. 不难看出
9. 值得注意的是
10. 不可忽视的是
11. 在当今社会
12. 随着科技的不断发展
13. 深入探讨
14. 旨在
15. 赋予
16. 助力
17. 此外、另外、与此同时、不仅如此

**替代方案：**
1. "本质上" → 删除，直接说核心观点
2. "说白了" → 删除，直接用口语表达
3. "不是...而是..." → 直接说是什么
4. "众所周知" → 大家都知道
5. "综上所述" → 总的来说 / 说到底
6. "总而言之" → 总的来说
7. "由此可见" → 所以说
8. "不难看出" → 我们能发现
9. "值得注意的是" → 值得一提的是
10. "不可忽视的是" → 有一个事实是
11. "在当今社会" → 现在
12. "随着科技的不断发展" → 随着 XXX 的成熟
13. "深入探讨" → 聊聊
14. "旨在" → 为了
15. "赋予" → 给
16. "助力" → 帮助

## 快速参考

### 标点符号速查

| 语境 | 标点 | 示例 |
|-----|------|------|
| 中文 | 全角 | ，。！？：； |
| 英文/代码 | 半角 | ,.!?/:; |
| 中英文间 | 空格 | AI Agent |
| 省略号 | 两个字符 | …… |

### 段落规范速查

| 规范 | 要求 |
|-----|------|
| 段落长度 | 不超过 5 行 |
| 句子长度 | 长句（15-25字）与短句（5-10字）交替 |
| 相似句子 | 不连续 3 个以上相似长度 |

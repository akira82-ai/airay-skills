# Post Polish Guide

初稿生成后的三阶段质量保障系统：审稿评分 → 十步打磨 → 诚实评估。

---

## Stage 1: 四维审稿 + 评分

初稿生成后，从四个维度评估并打分。

### 评分维度

| 维度 | 权重 | 标准 |
|------|------|------|
| 逻辑 | 30% | 论点清晰、论据充分、推理合理、无逻辑跳跃 |
| 表达 | 25% | 语言流畅、无 AI 痕迹、口语化适度、不生硬 |
| 数据 | 25% | 数据准确、案例恰当、引用规范、有外部验证 |
| 结构 | 20% | 标题吸引、段落分明、首尾呼应、节奏感好 |

### 评分标准

| 分数 | 等级 | 行动 |
|------|------|------|
| 90-100 | 优秀 | 可直接发布 |
| 80-89 | 良好 | 小幅优化后发布 |
| 70-79 | 合格 | 需要修改 |
| <70 | 不合格 | 需要重写 |

### 用户交互点 1：展示评分

向用户展示：
1. 总分（如 82/100）+ 各维度分数
2. 主要扣分项（Top 3）
3. 用户选择：进入打磨 / 补充信息后重新审稿

---

## Stage 2: 十步打磨

依次执行以下十步，每步聚焦一个质量维度。

### Step 1: 清晰度

**核心：每个句子都能一次读懂。**

- [ ] 无歧义代词（this/that/it 指代不明）
- [ ] 无复杂嵌套从句（拆成短句）
- [ ] 技术术语首次出现时有解释
- [ ] 段落间无逻辑跳跃

> 改前：那种在之前类似项目中曾被使用过的方法，这次也展现了不错的效果。
> 改后：这个方法我们之前在类似项目里用过，效果不错，这次也一样。

### Step 2: 语气一致性

**核心：全文保持同一语气，不忽正式忽随意。**

- [ ] 选定一种语气并贯穿全文（专业/对话/幽默/叙事）
- [ ] 段落间无突然的语气切换
- [ ] 个人叙事时 "我" 的使用一致
- [ ] 专业分析时不插入口语化表达

> 改前：AI 正在重塑各行各业。说实话这变化太疯狂了。企业采用率已达 78%。
> 改后：AI 正在以空前的速度重塑各行各业。企业采用率已达 78%。

### Step 3: So What 检验

**核心：每个论断都要回答"为什么读者该关心"。**

- [ ] 每个段落都与读者的利益/痛点相连
- [ ] 无脱离读者的理论陈述
- [ ] 数据后面跟着"这意味着什么"
- [ ] 每个论证都有明确的指向

> 改前：远程办公自 2020 年以来增长了 300%。
> 改后：远程办公自 2020 年以来增长了 300%——这意味着你的竞争对手很可能已经在全球招人了，而你还在受限于本地人才。

### Step 4: 证据支撑

**核心：每个主张都有数据/案例/引用支撑。**

- [ ] 统计数据有来源（搜索结果、报告）
- [ ] 主张有具体案例支撑
- [ ] 专家观点有引用
- [ ] 无空洞断言

> 改前：大多数团队使用 AI 工具后效率都有提升。
> 改后：McKinsey 2025 年报告显示，67% 使用 AI 工具的团队报告效率提升 20-40%。

### Step 5: 具体性

**核心：模糊词汇替换为具体数字/场景。**

- [ ] 无"很多/大量/显著"而无数字
- [ ] 无"效果很好/提升很大"而无量化
- [ ] 时间引用具体（"2026 年 3 月"而非"最近"）
- [ ] 结果可衡量

| 模糊 | 具体 |
|------|------|
| 节省时间 | 每周节省 4 小时 |
| 很多用户 | 2,847 个团队 |
| 效果很快 | 14 天见效 |
| 简单易用 | 10 分钟完成配置 |
| 显著提升 | 转化率从 3% 提升到 12% |

### Step 6: 情感共鸣

**核心：痛点要具象化，用微故事和感官细节。**

- [ ] 痛点用具体场景描述（非抽象概念）
- [ ] 至少一个"顿悟时刻"或转折点
- [ ] 情感词汇自然使用（不刻意）
- [ ] 读者能把自己代入描述的情境

> 改前：Debug 对开发者来说很痛苦。
> 改后：凌晨 3 点盯着满屏红字，报错信息看不懂，线上服务还在宕机——每个开发者都经历过这种绝望时刻。

### Step 7: 零风险

**核心：消除行动障碍，添加信任信号。**

- [ ] 推荐方法的第一步极其简单
- [ ] 主动回应可能的反对意见
- [ ] 有信任信号（数据、资质、成功案例）
- [ ] 无"你只需做 X"而无操作说明

> 改前：开始使用 AI 工具提升你的工作流。
> 改后：从今天起，写邮件时让 AI 帮你润色开头——这一个改变就够了，你会立刻感受到区别。

### Step 8: 节奏 + 排版

**核心：长短句交替，段落透气，中文排版规范。**

- [ ] 长句（15-25 字）与短句（5-10 字）交替
- [ ] 段落不超过 5 行
- [ ] 不连续 3 个以上相似长度句子
- [ ] 适度使用反问句和感叹句
- [ ] 过渡自然，首尾衔接不生硬

**中文排版规范：**
- 中英文之间加空格（AI Agent 而非 AIAgent，2024 年 而非 2024年）
- 中文语境用全角标点（，。！？：；）
- 英文/代码周围用半角标点
- 省略号用 ……（两个省略号字符）
- 术语全文统一（选定一种后保持一致）

### Step 9: 标题 + 首尾

**核心：标题 2 秒抓人，开头 5 秒留住，结尾驱动行动。**

**标题公式（按内容选用）：**
1. 数字型："7 个让效率翻倍的 AI 提示词技巧"
2. 提问型："为什么你的 AI 总是写不出好文章？"
3. 对比型："从 3 小时到 20 分钟：AI 如何改变我的写作"
4. 悬念型："试了 50 款 AI 工具后，我只留了这 3 个"

**开头技巧：**
- 痛点开头：读者立刻代入
- 反直觉：挑战常识
- 数据冲击：有力数字
- 故事钩子："上个月，我……"

**结尾技巧：**
- 金句升华：一句让人记住的话
- 行动号召：明确的下一步
- 开放问题：引发思考
- 承诺：告诉读者按做会怎样

### Step 10: 峰终定律

**核心：高潮（矛盾/困难/卡点最集中的段落）和结尾决定读者对文章的记忆。**

- [ ] 识别全文"峰值"段落——冲突/困难/洞见最密集处，确保足够生动详细
- [ ] 峰值段落包含全文最有力的案例或论证
- [ ] 放大转折点周围的矛盾和困难
- [ ] 结尾呼应开头（首尾圆合）
- [ ] 最后一段要有力，不能虎头蛇尾
- [ ] 如果是叙事文，高潮必须是最详细的部分

> 改前（弱高潮）：后来我找到了正确的方法，效果很好。
> 改后（强高潮）：整整两周，我试了不下 20 种方法，每次满怀希望、每次铩羽而归。直到那天深夜，我偶然把两个毫不相关的技巧组合在一起——屏幕上的数据突然跳了 3 倍。反复验证 5 遍，确认没看错。那一刻，所有弯路都值了。

## 快速替换表

### AI 痕迹词汇（必须替换）

| AI 味 | 人话 |
|--------|------|
| 综上所述 | 总的来说 |
| 总而言之 | 说到底 |
| 由此可见 | 所以说 |
| 不难看出 | 我们能发现 |
| 众所周知 | 大家都知道 |
| 值得注意的是 | 值得一提的是 |
| 不可忽视的是 | 有一个事实是 |
| 在当今社会 | 现在 |
| 随着科技的不断发展 | 随着 XXX 的成熟 |
| 深入探讨 | 聊聊 |
| 旨在 | 为了 |
| 赋予 | 给 |
| 助力 | 帮助 |

**必须删除的僵硬连接词：** 此外、另外、与此同时、不仅如此

### Filler 词（必须删除）

very, really, extremely, incredibly, quite, rather, somewhat, just, actually, basically, essentially, literally, in order to, the fact that, it should be noted that

### 弱动词替换

| 弱 | 强 |
|----|-----|
| utilize | use |
| implement | set up, build |
| leverage | use, apply |
| facilitate | help, enable |
| innovative | new, original |
| robust | strong, thorough |
| seamless | smooth, easy |
| cutting-edge | modern, latest |
| empower | enable, let |
| optimize | improve, streamline |

### 口语化增强词（适当添加）

其实、说实话、不得不说、你可能不知道、这里有个细节、让我来告诉你

## Stage 3: 诚实评估 + 用户确认

### 用户交互点 2：打磨结果确认

向用户展示 Top 3-5 处关键修改的 before/after 对比。用户选择：
- 接受所有修改
- 对特定修改提出调整
- 重新生成特定段落

### 用户交互点 3：诚实评估

向用户展示：
1. **优点**（2-3 点）：这篇文章突出的地方
2. **不足**（可接受范围内）：还可以改进的地方
3. **整体判断**：能否在目标平台发表？
4. 用户确认终稿

## 平台适配策略

母稿打磨到 90+ 后，按平台特点适配。

### 适配流程

```
打磨后的母稿（90+ 分）
    ↓
[按平台适配]
    ├── 公众号：完整版，保留全部章节
    ├── 小红书：精简版，emoji + 步骤清单
    ├── X/Twitter：Thread 格式（280 字符/条）
    └── LinkedIn：专业语气 + 个人经历角度
```

### X/Twitter Thread 格式

- 每条 280 字符（中文约 140 字）
- 编号格式：1/🧵 2/🧵 ……
- 结构：Hook → 支撑点 → 案例/证据 → 结论 + CTA
- 每条不超过 2 个 hashtag
- 每条可独立阅读 + 与 thread 连贯

### LinkedIn 格式

- 1,300 字符，前 210 字符决定是否被展开
- 专业但不呆板，用 "I" 分享经历
- 开头用个人场景或数据冲击
- 结尾用讨论问题引导互动

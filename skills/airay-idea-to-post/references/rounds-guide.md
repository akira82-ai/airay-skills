# 追问轮次详解

本文档详细说明 `idea-to-post` 技能中每个轮次的目的、任务和评估标准。

---

## [初始搜索] 背景信息收集

### 解决什么问题？

避免在完全不了解背景的情况下开始追问，导致问题偏离主题或遗漏关键信息。

### 核心任务是什么？

1. **优先读取用户提供的参考内容**
   - 理解用户文档/聊天记录的核心信息
   - 提取关键词、时间线、关键人物

2. **识别核心概念**
   - 从用户灵感中提取2-3个核心概念
   - 识别专业术语、行业词汇

3. **多角度背景搜索**
   - 基本定义："什么是[概念]"
   - 最新趋势："[概念] 2024/2025 趋势"
   - 行业背景："[概念] 行业应用"

### 如何评估达成了任务？

- [ ] 已读取并理解用户提供的所有参考内容
- [ ] 提取了2-3个核心概念
- [ ] 对主题有了基本了解
- [ ] 收集了足够的背景素材

---

## 第 1-2 轮：方向锁定（0-20分）

### 解决什么问题？

避免写到最后才发现"偏题了"，在一开始就明确写作的核心方向。

### 核心任务是什么？

1. **具体场景锁定**
   - 什么时候发生的？（具体时间）
   - 什么场景下？（工作/生活/学习）
   - 有什么触发事件？

2. **核心观点明确**
   - 最想表达的一个观点是什么？
   - 用一句话总结

3. **目标受众确认**
   - 为谁而写？
   - 他们关心什么？
   - 用什么语言风格？

4. **参考素材收集**
   - 有文档/截图/聊天记录吗？
   - 有数据/案例来源吗？

5. **风格偏好确认**
   - 专业严谨 / 轻松幽默 / 故事驱动？

### 如何评估达成了任务？

- [ ] 具体场景清晰（时间/地点/人物）
- [ ] 核心观点用一句话能说清楚
- [ ] 目标受众明确
- [ ] 已收集或确认有参考素材
- [ ] 风格偏好已确定

📚 参考：`references/core-principles.md`（原则2：多选+开放）

---

## 第 3 轮：确定组织方式（20-40分）

### 解决什么问题？

避免"想到哪写到哪"，先确定文章的整体结构框架。

### 核心任务是什么？

1. **根据灵感特点选择组织方式**
   - 包含"故事/转折/冲突" → SCQA
   - 包含"为什么/初衷/使命" → 黄金圈
   - 包含"推广/转化/营销" → AIDA
   - 包含"产品/功能/卖点" → FBA
   - 其他/不确定 → 5W1H

2. **确认选择理由**
   - 为什么这个框架适合？
   - 预期效果是什么？

### 如何评估达成了任务？

- [ ] 选择了明确的组织方式
- [ ] 能说明为什么选择这个框架
- [ ] 用户认同这个选择

📚 参考：`references/determining-frameworks.md`

---

## 第 4 轮：确定框架（20-40分）

### 解决什么问题？

避免"大而无当"，将抽象的组织方式转化为具体的章节标题。

### 核心任务是什么？

1. **划分部分数量**
   - 想分几个部分？（3-5个）
   - 不宜过多，读者容易迷失

2. **拟定部分标题**
   - 每部分的标题是什么？
   - 标题要能概括该部分核心内容

3. **明确每部分要点**
   - 每部分大致讲什么？
   - 2-3句话概括

### 如何评估达成了任务？

- [ ] 划分了3-5个部分
- [ ] 每个部分都有清晰的标题
- [ ] 标题能概括该部分的核心内容
- [ ] 各部分之间有逻辑顺序

---

## [精准搜索] 基于框架验证观点

### 解决什么问题？

避免"闭门造车"，在填充内容前先验证框架的合理性：
- 你的观点是否站得住脚？
- 有没有外部数据/案例支持？
- 是否存在反直觉的证据需要考虑？
- 框架是否需要调整？

### 核心任务是什么？

基于已搭建的框架，对每个部分的核心观点进行反向搜索验证：

1. **优先搜索用户提供的参考内容**
   - 在用户文档/聊天记录中寻找验证证据
   - 提取用户提及的具体案例、数据

2. **框架观点反向搜索**（按优先级）
   - Tier 1-2: 官方文档、权威报告（验证技术/行业观点）
   - Tier 3: GitHub、社区讨论（验证实践案例）
   - Tier 4-5: 媒体文章、知识平台（验证趋势/数据）

3. **寻找两类证据**
   - ✓ 验证证据：支持你观点的数据/案例
   - ✗ 反驳证据：与你观点相反的数据/案例

### 如何评估达成了任务？

检查清单：
- [ ] 每个部分的核心观点都找到了验证证据（≥1个）
- [ ] 发现的反驳证据已分析（为什么存在？如何回应？）
- [ ] 优先使用了用户提供的参考内容
- [ ] 所有引用都标注了来源
- [ ] 根据搜索结果，框架已调整优化（如需要）

**输出示例：**
```
[第1部分：xxx观点]
✓ 验证证据：根据xxx报告，[数据]
✓ 反驳证据：有观点认为[反驳内容]，但实际上[回应]

[第2部分：xxx观点]
✓ 验证证据：在您提供的文档中，[具体引用]
```

**关键区别：**
- [精准搜索]：验证框架是否合理（40-60分阶段）
- 第5-6轮素材收集：填充具体内容（40-60分阶段）

📚 参考：`references/data-sources.md`

## 第 5-6 轮：素材收集（40-60分）

### 解决什么问题？

避免"只有骨架没有血肉"，为每个部分收集具体的案例、数据、细节。

### 核心任务是什么？

**按框架的每个部分逐一收集素材：**

1. **第1部分素材**
   - 具体案例？（时间、地点、人物、数字）
   - 有对比吗？（之前 vs 之后）
   - 当时的感受？

2. **第2部分素材**
   - 需要什么具体例子？
   - 有数据支撑吗？
   - 有对比场景吗？

3. **后续部分素材**
   - 依次为每个部分收集对应素材
   - 确保每个部分都有足够的"血肉"

### 如何评估达成了任务？

- [ ] 每个部分都有≥1个具体案例
- [ ] 案例包含时间/地点/人物/数字
- [ ] 有前后对比或场景对比
- [ ] 素材与该部分的核心观点对应

📚 参考：`references/questioning-modes.md`（可选）

---

## 第 7 轮：情感共鸣（60-80分）

### 解决什么问题？

避免"理性分析无温度"，挖掘真实的情绪和身体反应，让内容有"活人感"。

### 核心任务是什么？

1. **最强烈的情绪时刻**
   - 最挫败的时刻？当时什么反应？
   - 最惊讶的时刻？记得第一反应吗？
   - 最兴奋的时刻？身体有什么感觉？

2. **身体反应记录**
   - 拍大腿？
   - 长叹一口气？
   - 瞬间坐直？
   - 手心出汗？

3. **转折点捕捉**
   - 从"没用"到"太神了"的瞬间
   - 从"怀疑"到"相信"的时刻
   - 从"困惑"到"恍然大悟"的点

### 如何评估达成了任务？

- [ ] 捕捉到≥2个强烈情绪时刻
- [ ] 记录了具体的身体反应
- [ ] 有清晰的转折点描述
- [ ] 情感描述真实、不矫情

📚 参考：`references/questioning-modes.md`（可选）

---

## 第 8 轮：独特观点（60-80分）

### 解决什么问题？

避免"人云亦云"，挖掘反直觉、独特的理解，让内容有独特价值。

### 核心任务是什么？

1. **未发现的技巧**
   - 有大多数人不知道的用法吗？
   - 有意外的发现吗？

2. **独特的用法**
   - 和常规用法不同？
   - 组合使用的创新？

3. **反直觉的理解**
   - 与大众认知相反的观点？
   - 为什么会有这种理解？

4. **可引用的原话**
   - 有文档/对话的原话吗？
   - 有金句吗？

### 如何评估达成了任务？

- [ ] 有≥1个独特观点或技巧
- [ ] 有反直觉的理解
- [ ] 有可引用的原话/金句
- [ ] 观点有依据，不是哗众取宠

---

## 第 9 轮：初稿生成 + 四维审稿（80-100分）

### 解决什么问题？

避免"盲目成稿"，先展示初稿并评估质量，根据反馈进行针对性优化。

### 核心任务是什么？

1. **整合所有信息生成初稿**
   - 核心观点
   - 各部分素材
   - 情感共鸣
   - 独特观点
   - 搜索验证

2. **四维评分**
   - 逻辑（30%）：观点是否清晰，论证是否严密
   - 表达（25%）：语言是否流畅，是否有"活人感"
   - 数据（25%）：是否有足够的案例/数据支撑
   - 结构（20%）：结构是否清晰，各部分是否连贯

3. **展示扣分项**
   - 哪里扣分了？为什么？
   - 具体的问题是什么？

4. **用户反馈**
   - 需要补充什么？
   - 哪里不满意？

### 如何评估达成了任务？

- [ ] 生成了完整的初稿
- [ ] 完成了四维评分
- [ ] 明确展示了扣分项
- [ ] 收集了用户反馈

📚 参考：`references/post-polish.md`

---

## 第 10 轮：深度打磨与确认（80-100分）

### 解决什么问题？

避免"小修小补"，进行系统性的深度打磨，确保达到优质内容标准。

### 核心任务是什么？

1. **根据第9轮反馈补充内容**
   - 补充缺失的案例
   - 强化情感描述
   - 突出独特观点

2. **十步深度打磨**
   - 开头是否吸引人？
   - 标题是否准确？
   - 段落是否过长？
   - 是否有AI味？
   - 案例/数据是否足够？
   - 逻辑是否严密？
   - 结尾是否有力？
   - 格式是否统一？
   - 是否有错别字？
   - 整体是否有"活人感"？

3. **诚实评估**
   - 优点：3点
   - 不足：3点
   - 能否发表？（是/否/需修改）

### 如何评估达成了任务？

- [ ] 完成了十步打磨
- [ ] 提供了诚实评估（优点+不足）
- [ ] 明确说明能否发表
- [ ] 用户确认终稿

📚 参考：`references/post-polish.md`

## 轮次关系图

```
[初始搜索] 背景了解
    ↓
第 1-2 轮：方向锁定（0-20分）—— 明确写什么、为谁写
    ↓
第 3 轮：确定组织方式（20-40分）—— 选择整体框架
    ↓
第 4 轮：确定框架（20-40分）—— 划分章节标题
    ↓
[精准搜索] 验证观点 —— 验证框架合理性
    ↓
第 5-6 轮：素材收集（40-60分）—— 填充具体内容
    ↓
第 7 轮：情感共鸣（60-80分）—— 挖掘情绪反应
    ↓
第 8 轮：独特观点（60-80分）—— 提炼独特价值
    ↓
第 9 轮：初稿 + 审稿（80-100分）—— 生成并评估
    ↓
第 10 轮：深度打磨（80-100分）—— 优化到优质
```

---

## 关键原则

1. **渐进深化**：每轮都基于上一轮的答案
2. **框架内化**：用框架思维设计问题，但不暴露框架名称
3. **多选+开放**：快速锁定方向 + 深度挖掘内容
4. **必须完整**：案例、情感、独特性、原始素材引用、故事闭环都是必填项

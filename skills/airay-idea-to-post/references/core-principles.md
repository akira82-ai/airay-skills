# 核心原则

## 原则 1：内化框架，不暴露框架

### 设计思想

思维框架是辅助思考的工具，不是机械套用的模板。如果直接告诉用户"我正在使用 XX 框架"，会让对话变得生硬、机械。

### 具体做法

- 用框架逻辑来设计问题，但用自然语言表达
- 问题要有框架思维支撑，但听起来像日常对话
- 框架是后台的"思考地图"，不是前台的话术脚本

### 正确示例

```
❌ 错误："我推荐使用 PREP 框架。现在第一步 Point：你的观点是什么？"
✅ 正确："你想表达的核心观点是什么？"
```

### 效果

- 用户感觉在自然对话，而非填问卷
- 问题逻辑清晰，层层递进
- 框架内化后，可以灵活调整顺序

## 原则 2：多选题 + 开放题结合

### 设计思想

- **多选题**：快速锁定方向，减少认知负担
- **开放题**：深度挖掘内容，获取丰富素材
- 两者结合，既有结构，又有深度

### 具体做法

**多选题使用时机：**
- 需要快速分类（目标受众、文章类型、风格偏好）
- 有明确的几个选项
- 用户需要灵感启发

**开放题使用时机：**
- 需要故事、经历、情感
- 需要具体细节（时间、地点、人物、数字）
- 需要深入挖掘"为什么"

### 混合使用技巧

```
多选题后接开放题：
"你希望文章偏向哪种风格？
A. 专业严谨  B. 轻松幽默  C. 故事驱动
（用户选择后）
→ 能具体说说你为什么想选这种风格吗？有什么参考吗？"
```

### 效果

- 多选题 = 快速定位，搭建骨架
- 开放题 = 深度挖掘，填充血肉

## 原则 3：渐进深化，动态调整

### 设计思想

每轮问题都基于上一轮的回答，像剥洋葱一样层层深入，而不是机械地按固定顺序提问。

### 具体做法

**自然过渡技巧：**
1. 抓住上轮回答中的关键词或疑问点
2. 从一个维度自然过渡到相关维度
3. 根据用户回答的丰富度，决定追问深度

### 实战示例

```
用户："todo 是一个被低估的命令"
    ↓
追问："被低估"具体指什么？（概念深化）
    ↓
用户："人们不知道它是对话记忆机制"
    ↓
追问：它解决了什么痛点？（价值探寻）
    ↓
追问：有什么具体例子？（案例补充）
    ↓
用户：上次我忘记重要事项...
    ↓
追问：当时什么感受？（情感挖掘）
```

### 动态调整原则

- 用户回答简单 → 追问要具体
- 用户回答丰富 → 可以跳跃到下一维度
- 用户提到新角度 → 灵活转向
- 用户情绪激动 → 先共情，再追问

### 效果

- 对话自然流畅，不生硬
- 每轮都有价值，逐步深入
- 用户感觉被理解，而非被审问

## 原则 4：连续追问（5-Why）

### 设计思想

对关键问题连续追问"为什么"，层层深入直到找到根本原因。这是"渐进深化"原则的具体体现。

### 具体做法

**五层追问法：**
1. 第一层：表面现象是什么？
2. 第二层：为什么会出现这个问题？
3. 第三层：背后的原因是什么？
4. 第四层：更深层的原因是什么？
5. 第五层：根本原因是什么？

**何时使用：**
- 用户提到"问题"、"挑战"、"困惑"时
- 想挖掘深层原因时
- 想找到可行动的解决方案时

### 实战示例

```
用户："我们的用户留存率一直很低"
    ↓
追问："为什么低？" → 用户用一次就不回来了
    ↓
追问："为什么不用了？" → 找不到核心功能
    ↓
追问："为什么找不到？" → 产品导航太复杂
    ↓
追问："为什么复杂？" → 我们在做加法不做减法
    ↓
追问："为什么不做减法？" → 缺乏"少即是多"的设计理念（根本原因）
```

### 效果

- 找到根本原因，而不是停留在表面
- 每一层都比前一层更有价值
- 最终找到可行动的解决方案

## 原则 5：观点表达逻辑（PREP）

### 设计思想

观点表达应该遵循逻辑顺序：Point → Reason → Example → Point。这是让表达清晰有力的基本原则。

### 具体做法

**四个步骤：**
1. **Point（观点）** - 直接陈述核心观点
2. **Reason（理由）** - 解释为什么持有这个观点
3. **Example（案例）** - 用具体案例支撑
4. **Point（总结）** - 重申观点，强化印象

**何时使用：**
- 用户表达观点时
- 收集案例时
- 让用户阐述理由时

### 实战示例

```
❌ 错误的追问顺序：
"有什么案例吗？你的观点是什么？为什么这么想？"

✅ 正确的追问顺序（PREP）：
"核心观点是什么？"（Point）
    ↓
"为什么这么认为？"（Reason）
    ↓
"有什么具体案例吗？"（Example）
    ↓
"所以，这个观点是..."（Point 总结）
```

### 效果

- 表达逻辑清晰，层次分明
- 观点、理由、案例环环相扣
- 让表达更有说服力

## 原则 6：深度思考（第一性原理）

### 设计思想

从最基本的事实出发，分解到本质，然后重构解决方案。这是"创新"和"深度思考"的核心方法。

### 具体做法

**三步法：**
1. **分解** - 将问题分解到最基本的事实/真理
2. **质疑** - 质疑所有假设和"常识"
3. **重构** - 从基本原理重建解决方案

**何时使用：**
- 用户提到"创新"、"突破"、"颠覆"时
- 想挖掘反直觉观点时
- 想打破常规思维时

### 实战示例

```
用户："大家都说这个方案不可行"
    ↓
追问："这个方案的本质是什么？"（分解）
    ↓
追问："真的不可行吗？基于什么判断？"（质疑）
    ↓
追问："如果打破常规，有没有新方法？"（重构）
```

### 效果

- 打破"大家都这么说"的思维陷阱
- 找到反直觉的独特观点
- 从第一性原理出发，而不是类比

## 原则 7：段落节奏（长短句交替）

### 设计思想

好的段落节奏让文章易读、有呼吸感。读者在屏幕上阅读时，短段落和长短句交替能减轻阅读疲劳。

### 具体做法

**段落长度控制：**
- 每段不超过 5 行
- 长句（15-25 字）与短句（5-10 字）交替
- 不连续 3 个以上相似长度句子

**分段时机：**
1. **观点切换时** - 一个观点表达完毕，立即分段
2. **场景转换时** - 时间/地点/人物变化时分段
3. **强调重点时** - 独立成段，突出关键信息
4. **对话引用时** - 对话前后各空一行
5. **超过 5 行时** - 强制分段，保持视觉透气

**段落过渡：**
- 用过渡词连接（但是/因此/然而）
- 用问句引出下一段
- 用总结句收尾上段，再开启新段

### 实战示例

```
❌ 错误：长段落堆叠
在当今社会，随着科技的不断发展，人工智能正在改变我们的生活方式。众所周知，AI 已经在很多领域展现出了强大的能力，其中包括自然语言处理、计算机视觉、机器学习等等。值得注意的是，这些技术的应用范围正在不断扩大，深入到各行各业。不可忽视的是，AI 带来的便利同时也伴随着一些挑战。综上所述，我们应该理性看待 AI 的发展。

✅ 正确：长短句交替，段落透气
AI 正在改变我们的生活。

这不是未来时态，而是正在发生的事实。

从自然语言处理到计算机视觉，AI 在多个领域展现出了强大的能力。应用范围也在不断扩大，深入到各行各业。

但便利背后也有挑战。

---

## 总结

七大原则相辅相成：

1. **内化框架** → 保证问题有逻辑
2. **多选+开放** → 兼顾效率和深度
3. **渐进深化** → 对话自然流畅
4. **连续追问（5-Why）** → 层层深入，找到根本原因
5. **观点表达逻辑（PREP）** → 表达清晰有力
6. **深度思考（第一性原理）** → 打破常规，创新突破
7. **段落节奏（长短句交替）** → 易读有呼吸感

核心目标：让 AI 像真实的访谈者一样，引导用户表达出深度的、有故事性的内容。

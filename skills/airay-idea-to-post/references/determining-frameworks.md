# 组织框架详解

本文档提供 `idea-to-post` 技能中可使用的**文章组织方式**框架的详细说明。

**注意**：核心原则（5-Why、PREP、第一性原理等）请参考 `core-principles.md`。

---

## 1. 5W1H 分析法

### 概述

5W1H 是最全面的信息收集框架，确保覆盖问题的所有维度。

### 六个维度

| 维度 | 问题 | 目的 |
|-----------|----------|---------|
| What | 是什么？ | 明确核心概念/问题/产品 |
| Why | 为什么？ | 探索原因、动机、意义 |
| Who | 谁？ | 识别主体、受众、利益相关者 |
| When | 什么时候？ | 时间维度、时机选择 |
| Where | 在哪里？ | 空间维度、场景定位 |
| How | 怎么做？ | 方法、路径、实施 |

### 适用场景

- 需要全面介绍一个主题
- 项目启动和规划
- 复杂问题分解和分析
- 不确定使用哪个框架时的默认选择

### 文章结构示例

```
[What-是什么] 什么是AI？一个简单的定义

[Why-为什么] AI为什么重要？它带来的变化

[Who-谁] 谁在使用AI？从个人到企业

[When-什么时候] 现在是AI时代吗？时机分析

[Where-在哪里] AI应用在哪些领域？

[How-如何] 如何开始使用AI？
```

## 2. SCQA 叙事模型

### 概述

SCQA 是麦肯锡顾问常用的叙事结构，非常适合故事型内容。

### 四个阶段

| 阶段 | 含义 | 内容 |
|-------|---------|---------|
| S (Situation) | 情境 | 描述当前稳定状态/背景 |
| C (Complication) | 冲突 | 出现的问题/变化/挑战 |
| Q (Question) | 问题 | 如何解决这个问题？ |
| A (Answer) | 答案 | 提供解决方案/洞察 |

### 适用场景

- 讲述从0到1的产品/项目故事
- 分享解决问题的经历
- 职场成长故事
- 创业历程分享

### 文章结构示例

```
[S-情境] 一切看起来很正常，我们按部就班地工作...

[C-冲突] 直到有一天，用户投诉激增30%，系统崩溃...

[Q-问题] 为什么会出现这个问题？应该如何应对？

[A-答案] 经过深入分析，我们找到了根本原因...并找到了解决方案...
```

### 使用技巧

- 让情境与读者产生共鸣
- 冲突要有戏剧性，制造张力
- 问题应该自然引出答案
- 答案应该出人意料或发人深省

## 3. 黄金圈法则 (Golden Circle)

### 概述

Simon Sinek 提出的经典模型，从内向外思考：Why → How → What

### 三个层次

| 层次 | 问题 | 描述 |
|-------|----------|-------------|
| Why (核心) | 为什么？ | 信念、初衷、使命、存在的意义 |
| How (中间) | 怎么做？ | 方法论、过程、实施方式 |
| What (外层) | 是什么？ | 具体的产品、服务、结果 |

### 适用场景

- 品牌故事和理念表达
- 产品价值主张
- 个人/团队使命介绍
- 创业初衷分享

### 文章结构示例

```
[Why-为什么] 我们相信每个人都应该有能力控制自己的数据...

[How-如何] 因此我们构建了一个端到端加密的去中心化存储系统...

[What-是什么] 最终呈现给用户的产品是一个简单、安全的个人云盘...
```

### 使用技巧

- 从 Why 开始 - 这是最强有力的部分
- How 应该展示独特的方法论
- What 是结果，但不是重点
- 大多数人从 What 开始，从 Why 开始让你脱颖而出

## 4. AIDA 营销模型

### 概述

经典营销漏斗模型，引导读者从注意到行动。

### 四个阶段

| 阶段 | 目标 | 方法 |
|-------|------|--------|
| Attention | 吸引注意 | 震撼数据、反直觉观点、悬念 |
| Interest | 激发兴趣 | 相关性、痛点共鸣 |
| Desire | 创造欲望 | 展示利益、社会证明 |
| Action | 促使行动 | 明确的行动指引、紧迫感 |

### 适用场景

- 产品推广
- 课程/活动推广
- 转化型文案
- 营销活动

### 文章结构示例

```
[Attention-注意] 90%的人学编程的方法都错了...

[Interest-兴趣] 你花了很多时间但进展缓慢，不是你不聪明...

[Desire-欲望] 想象一下如果有一种方法能帮你在3个月内独立做项目...

[Action-行动] 点击链接免费获取我的编程学习路线图...
```

### 使用技巧

- Attention 要足够震撼
- Interest 要触动痛点
- Desire 要描绘美好愿景
- Action 要具体明确

## 5. FBA 产品介绍法

### 概述

经典的产品营销结构，从功能到利益到优势。

### 三个层次

| 层次 | 内容 | 示例 |
|-------|---------|---------|
| Feature | 功能 | 产品有什么功能 |
| Benefit | 利益 | 这个功能给用户带来什么 |
| Advantage | 优势 | 与其他产品有什么不同 |

### 适用场景

- 产品介绍
- 功能发布
- 服务描述
- 对比评测

### 文章结构示例

```
[Feature-功能] 我们的笔记软件支持双向链接

[Benefit-利益] 这意味着你可以建立知识网络，让想法连接，发现隐藏的关联...

[Advantage-优势] 与传统笔记软件的文件夹结构不同，这更像大脑的工作方式...
```

### 使用技巧

- Feature 要简洁具体
- Benefit 要从用户角度出发（"你能得到什么"）
- Advantage 要突出差异化
- 可以有多个 Feature-Benefit-Advantage 组合

## 框架选择决策树

```
你的灵感
    │
    ├─ 包含"为什么/本质/初衷/使命/价值" → 黄金圈法则
    ├─ 包含"问题/挑战/困境/转折点/故事" → SCQA
    ├─ 包含"推广/宣传/转化/销售/营销" → AIDA
    ├─ 包含"产品/功能/优势/卖点/特征" → FBA
    └─ 其他/不清楚 → 5W1H
```

---

## 框架组合使用

对于复杂内容，可以组合多个框架：

- **5W1H + SCQA**：先全面了解背景，再讲述故事
- **黄金圈 + FBA**：先清晰阐述理念，再介绍产品
- **5W1H + AIDA**：先收集信息，再吸引转化

不要被框架限制 - 框架是工具，不是目标。
